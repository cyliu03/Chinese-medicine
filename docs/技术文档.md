## 1. 项目概述

### 1.1 项目背景

中医方剂学是中医药学的核心组成部分，研究方剂的组成、变化和临床运用规律。传统方剂推荐依赖于中医师的个人经验和知识积累，存在以下问题：

- **知识传承困难**：名老中医经验难以系统化传承
- **诊断效率低下**：年轻医师需要大量时间学习和积累
- **标准化程度低**：不同医师对同一症状组合可能给出不同方剂

### 1.2 项目目标

本项目旨在构建一个基于深度学习的中医方剂推荐系统，实现：

| 目标 | 描述 | 验收标准 |
|------|------|----------|
| 准确性 | 正确推荐方剂药材 | F1 ≥ 0.90 |
| 完整性 | 不遗漏关键药材 | Recall ≥ 0.90 |
| 精确性 | 不多开无关药材 | Precision ≥ 0.90 |
| 可解释性 | 提供推荐理由 | 注意力权重可视化 |

### 1.3 技术选型

| 模块 | 技术栈 | 版本要求 |
|------|--------|----------|
| 深度学习框架 | PyTorch | ≥ 2.0 |
| 后端服务 | FastAPI | ≥ 0.100 |
| 前端框架 | Next.js | 14 |
| 编程语言 | Python | ≥ 3.8 |
| 容器化 | Docker | ≥ 20.0 |

#### 3.3.1 增强策略

原始数据为40个经典方剂，经过数据增强扩充到8,600个样本：

**策略一：症状同义词替换**

```python
def synonym_replacement(symptoms, replace_prob=0.3):
    """
    以一定概率将症状替换为同义词
    
    示例：
    输入: ["咳嗽", "发热", "头痛"]
    输出: ["咳痰", "发烧", "头痛"]  # 部分替换
    """
    new_symptoms = []
    for s in symptoms:
        if s in SYMPTOM_VARIATIONS and random.random() < replace_prob:
            new_symptoms.append(random.choice(SYMPTOM_VARIATIONS[s]))
        else:
            new_symptoms.append(s)
    return new_symptoms
```

**策略二：症状子集采样**

```python
def symptom_subset_sampling(symptoms, min_keep=3):
    """
    随机保留部分症状，模拟临床诊断中患者描述不完整的情况
    
    示例：
    输入: ["恶寒", "发热", "无汗", "头痛", "身痛", "喘"]
    输出: ["恶寒", "发热", "无汗", "头痛"]  # 随机保留4个
    """
    if len(symptoms) <= min_keep:
        return symptoms.copy()
    k = random.randint(min_keep, len(symptoms))
    return random.sample(symptoms, k)
```

**策略三：症状顺序打乱**

```python
def shuffle_symptoms(symptoms):

    打乱症状顺序，使模型不依赖于输入顺序
```

```python
    random.shuffle(shuffled_data)

    train_end = int(n * train_ratio)
```

## 4. 算法设计

### 4.1 问题定义

#### 4.1.1 任务类型

本项目是一个**多标签分类问题**（Multi-label Classification）：

```
输入: 症状集合 X = {s₁, s₂, ..., sₙ}
输出: 药材集合 Y = {h₁, h₂, ..., hₘ}
```

#### 4.1.2 与传统分类的区别

| 特性 | 单标签分类 | 多标签分类（本项目） |
|------|------------|----------------------|
| 输出类别数 | 1个 | 多个（0到全部） |
| 标签关系 | 互斥 | 可共存、相关 |
| 损失函数 | CrossEntropy | BCEWithLogitsLoss |
| 输出层激活 | Softmax | Sigmoid |
| 评估指标 | Accuracy | Precision, Recall, F1 |

#### 4.1.3 数学建模

给定症状向量 $\mathbf{x} \in \{0,1\}^{|V_s|}$，预测药材标签集合 $\mathbf{y} \in \{0,1\}^{|V_h|}$：

$$P(y_j=1|\mathbf{x}) = \sigma(f_j(\mathbf{x}))$$

其中 $\sigma$ 为sigmoid函数，$f_j(\mathbf{x})$ 为神经网络对第 $j$ 个药材的输出。

### 4.3 核心组件详解

#### 4.3.1 标签特定注意力

**设计动机**：不同药材应该关注不同的症状组合

```
麻黄应该关注: 恶寒、无汗、喘
甘草应该关注: 乏力、气短、需要调和
黄连应该关注: 口苦、心烦、舌红
```

**实现代码**：

```python
class LabelSpecificAttention(nn.Module):
    def __init__(self, hidden_dim, label_dim):
        super().__init__()
        self.attention = nn.Sequential(
            nn.Linear(hidden_dim + label_dim, hidden_dim // 2),
            nn.Tanh(),
            nn.Linear(hidden_dim // 2, 1),
        )
    
    def forward(self, x, label_embed):
        """
        参数:
            x: 症状编码 [batch, hidden_dim]
            label_embed: 标签嵌入 [batch, num_labels, label_dim]
        
        返回:
            label_features: [batch, num_labels, hidden_dim]
        """
        batch_size, num_labels, _ = label_embed.shape
        
        # 扩展症状编码
        x_expanded = x.unsqueeze(1).expand(-1, num_labels, -1)
        
        # 拼接特征
        combined = torch.cat([x_expanded, label_embed], dim=-1)
        
        # 计算注意力权重
        attention_weights = self.attention(combined).squeeze(-1)
        attention_weights = F.softmax(attention_weights, dim=-1)
        
        # 加权求和
        label_features = torch.einsum('bl,bld->bld', attention_weights, x_expanded)
        
        return label_features
```

#### 4.3.2 标签相关性建模

**设计动机**：药材之间存在配伍关系

```
相须配伍: 麻黄 + 桂枝 → 增强发汗解表
相使配伍: 大黄 + 芒硝 → 峻下热结
调和诸药: 甘草调和诸药，缓解药物烈性
```

**实现代码**：

```python
class LabelCorrelation(nn.Module):
    def __init__(self, hidden_dim, num_heads=8, dropout=0.3):
        super().__init__()
        self.self_attention = nn.MultiheadAttention(
            embed_dim=hidden_dim,
            num_heads=num_heads,
            dropout=dropout,
            batch_first=True
        )
        self.norm = nn.LayerNorm(hidden_dim)
    
    def forward(self, label_features):
        """
        参数:
            label_features: [batch, num_labels, hidden_dim]
        
        返回:
            correlated_features: [batch, num_labels, hidden_dim]
        """
        # 自注意力
        attn_output, _ = self.self_attention(
            query=label_features,
            key=label_features,
            value=label_features
        )
        
        # 残差连接 + 层归一化
        output = self.norm(label_features + attn_output)
        
        return output
```

### 4.4 损失函数设计

#### 4.4.1 设计动机

在中医方剂推荐中，漏开药材比多开药材更严重：

| 错误类型 | 后果 | 严重程度 |
|----------|------|----------|
| 漏开君药 | 方剂失去主要治疗作用 | 严重 |
| 漏开臣药 | 疗效大打折扣 | 较严重 |
| 多开调和药 | 增加成本，影响较小 | 较轻 |
| 多开功效相近药材 | 可能增强疗效 | 轻微 |

#### 4.4.2 非对称损失函数

$$L = -\alpha_{fn} \sum_{i} y_i \log(\hat{y}_i) - \alpha_{fp} \sum_{i} (1-y_i) \log(1-\hat{y}_i)$$

其中：
- $\alpha_{fn} = 0.85$：漏开惩罚权重
- $\alpha_{fp} = 0.15$：多开惩罚权重

**实现代码**：

```python
class AsymmetricLoss(nn.Module):
    def __init__(self, alpha_fn=0.85, alpha_fp=0.15):
        super().__init__()
        self.alpha_fn = alpha_fn
        self.alpha_fp = alpha_fp
    
    def forward(self, logits, targets):
        probs = torch.sigmoid(logits)
        
        # 漏开损失 (y=1时)
        fn_loss = -self.alpha_fn * targets * torch.log(probs + 1e-7)
        
        # 多开损失 (y=0时)
        fp_loss = -self.alpha_fp * (1 - targets) * torch.log(1 - probs + 1e-7)
        
        return (fn_loss + fp_loss).mean()
```

### 4.5 评价指标

#### 4.5.1 指标定义

**Precision（精确率）**：

$$Precision = \frac{TP}{TP + FP}$$

含义：预测为有的药材中，真正应该有的比例。

**Recall（召回率）**：

$$Recall = \frac{TP}{TP + FN}$$

含义：实际应有的药材中，被正确预测的比例。

**F1 Score**：

$$F1 = \frac{2 \times Precision \times Recall}{Precision + Recall}$$

含义：Precision和Recall的调和平均数。

#### 4.5.2 计算示例

```
预测药材: 麻黄, 桂枝, 杏仁, 黄芩     (4味)
实际药材: 麻黄, 桂枝, 杏仁, 甘草    (4味)

TP = 3 (麻黄, 桂枝, 杏仁)
FP = 1 (黄芩 - 多开)
FN = 1 (甘草 - 漏开)

Precision = 3 / 4 = 0.75
Recall = 3 / 4 = 0.75
F1 = 2 * 0.75 * 0.75 / (0.75 + 0.75) = 0.75
```

```python
class EarlyStopping:
    def __init__(self, patience=100, min_delta=0.0001):
        self.patience = patience
        self.min_delta = min_delta
        self.counter = 0
        self.best_score = None
    
    def __call__(self, score):
        if self.best_score is None:
            self.best_score = score
            return False
        
        if score > self.best_score + self.min_delta:
            self.best_score = score
            self.counter = 0
        else:
            self.counter += 1
        
        return self.counter >= self.patience
```

### 5.3 训练流程

```python
def train_epoch(model, loader, criterion, optimizer, device, scaler):
    model.train()
    total_loss = 0
    metrics = {'tp': 0, 'fp': 0, 'fn': 0}
    
    for symptoms, herbs in loader:
        symptoms = symptoms.to(device)
        herbs = herbs.to(device)
        
        optimizer.zero_grad()
        
        # 混合精度训练
        with torch.amp.autocast('cuda'):
            logits, _ = model(symptoms)
            loss = criterion(logits, herbs)
        
        scaler.scale(loss).backward()
        scaler.step(optimizer)
        scaler.update()
        
        total_loss += loss.item()
        
        # 计算指标
        probs = torch.sigmoid(logits)
        preds = (probs > 0.5).float()
        metrics['tp'] += (preds * herbs).sum().item()
        metrics['fp'] += (preds * (1 - herbs)).sum().item()
        metrics['fn'] += ((1 - preds) * herbs).sum().item()
    
    precision = metrics['tp'] / (metrics['tp'] + metrics['fp'] + 1e-7)
    recall = metrics['tp'] / (metrics['tp'] + metrics['fn'] + 1e-7)
    f1 = 2 * precision * recall / (precision + recall + 1e-7)
    
    return total_loss / len(loader), {'precision': precision, 'recall': recall, 'f1': f1}
```
